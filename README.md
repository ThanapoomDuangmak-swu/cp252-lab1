# Requirement document
## Functional requirement
1. ต้องแสดงสถานะช่องจอดว่าง Real-time แยกตามโซนหรือชั้นได้  
2. จองที่จอดรถล่วงหน้าโดยระบุวัน เวลา และสถานที่ได้  
3. ยกเลิกการจองและเปลี่ยนสถานะเป็นว่างทันทีเมื่อผู้ใช้ไม่มาตามเวลาที่กำหนด  
4. สามารถกดยกเลิกการจองได้ด้วยตนเองผ่านแอพพลิเคชัน  
5. คำนวณค่าบริการจอดรถตามระยะเวลาจริงและอัตราที่ห้างกำหนดได้  
6. ผู้ดูแลต้องสามารถเพิ่ม ลบ หรือแก้ไขผังที่จอดรถผ่านระบบจัดการได้  
7. ตรวจสอบและแสดงยอดรวมจำนวนรถที่จอดอยู่ทั้งหมดในห้างได้  
8. ระบบต้องมีการยืนยันตัวตน เช่น QR Code หรือ ทะเบียนรถ เมื่อนำรถเข้าจอด  
9. บันทึกข้อมูลส่วนตัวและทะเบียนรถเพื่อใช้ในการจองได้  
10. มีการแจ้งเตือนสถานะการจองและการหมดเวลาจองไปบอกผู้ใช้งาน  
## Non-Functional requirement
1. ต้องรองรับการใช้งานบนมือถือทั้ง Android และ iOS  
2. ระบบต้องมีเมนูภาษาไทยทั้งหมดเพื่อให้ผู้ใช้งานทั่วไปเข้าใจได้สะดวก  
3. แอพพลิเคชันต้องทำงานได้รวดเร็ว กดแล้วเปลี่ยนหน้าทันที ไม่ค้างหรือหน่วง  
4. ระบบต้องเปิดให้เข้าใช้งานและจองได้ 24 ชั่วโมง  
5. ตัวหนังสือและปุ่มกดต้องมองเห็นชัดเจน ใช้งานง่ายแม้กับผู้สูงอายุ  


## ข้อกำหนดคุณสมบัติระบบบริหารจัดการที่จอดรถอัจฉริยะ
### 1. ระบบแสดงผลสำหรับผู้ใช้งาน (Front-End)  
1.1 ระบบต้องแสดงสถานะช่องจอดรถแบบ Real-time โดยสามารถแยกการแสดงผลตาม  
-โซนที่จอดรถ  
-ชั้นของอาคารจอดรถ  
1.2 ระบบต้องมีหน้าจอแสดงผังที่จอดรถ (Parking Layout) ที่ผู้ใช้งานสามารถดูตำแหน่งช่องจอดว่างและไม่ว่างได้อย่างชัดเจน  
1.3 ผู้ใช้งานสามารถทำการ จองที่จอดรถล่วงหน้า โดยระบุ  
-วันที่  
-เวลาเริ่มต้นและสิ้นสุด  
-สถานที่หรือโซนที่ต้องการจอด  
1.4 ผู้ใช้งานสามารถ ยกเลิกการจองด้วยตนเอง ผ่านแอปพลิเคชันได้ตลอดเวลา  
1.5 ระบบต้องมีการแจ้งเตือนสถานะการจอง เช่น  
-แจ้งเตือนเมื่อจองสำเร็จ  
-แจ้งเตือนก่อนหมดเวลาจอง  
-แจ้งเตือนเมื่อการจองถูกยกเลิก  
1.6 ระบบต้องรองรับการยืนยันตัวตนของผู้ใช้งานเมื่อนำรถเข้าจอด โดยใช้วิธีใดวิธีหนึ่งหรือหลายวิธีร่วมกัน เช่น  
-QR Code  
-หมายเลขทะเบียนรถ  
1.7 ระบบต้องบันทึกข้อมูลส่วนตัวของผู้ใช้งาน เช่น  
-ชื่อ–นามสกุล  
-หมายเลขทะเบียนรถ  
เพื่อใช้ในการจองและยืนยันตัวตน  
### 2. ระบบจัดการการจองและการใช้งานที่จอดรถ  
2.1 ระบบต้องเปลี่ยนสถานะช่องจอดรถเป็น ว่างทันที หากผู้ใช้งานไม่มาตามเวลาที่จองไว้ภายในเวลาที่กำหนด  
2.2 ระบบต้องสามารถคำนวณค่าบริการจอดรถตาม  
-ระยะเวลาการจอดจริง  
-อัตราค่าบริการที่ห้างหรือสถานที่กำหนด  
2.3 ระบบต้องสามารถแสดงรายละเอียดค่าบริการให้ผู้ใช้งานตรวจสอบได้ก่อนและหลังการจอดรถ  
2.4 ระบบต้องสามารถตรวจสอบและแสดง ยอดรวมจำนวนรถที่จอดอยู่ทั้งหมด ภายในห้างหรือพื้นที่จอดรถได้แบบ Real-time  
### 3. ระบบบริหารจัดการสำหรับผู้ดูแลระบบ (Back-End)  
3.1 ระบบต้องมีหน้าจอสำหรับผู้ดูแลระบบ (Admin) เพื่อใช้ในการจัดการข้อมูลต่าง ๆ  
3.2 ผู้ดูแลระบบต้องสามารถ  
-เพิ่มผังที่จอดรถ  
-แก้ไขผังที่จอดรถ  
-ลบผังที่จอดรถ  
3.3 ผู้ดูแลระบบสามารถกำหนด  
-จำนวนช่องจอด  
-โซนหรือชั้น  
-สถานะเริ่มต้นของช่องจอดรถ  
3.4 ระบบต้องสามารถแสดงรายงานสถิติ เช่น  
-จำนวนการจอง  
-จำนวนรถที่เข้าจอดในแต่ละช่วงเวลา  
-อัตราการใช้งานที่จอดรถ  
### 4. การพัฒนาในรูปแบบ Mobile Application  
4.1 ระบบต้องรองรับการใช้งานบนอุปกรณ์  
-Android  
-iOS  
4.2 แอปพลิเคชันต้องออกแบบให้รองรับการใช้งานบนหน้าจอมือถือ (Mobile Friendly)  
4.3 เมนูและการใช้งานทั้งหมดต้องเป็น ภาษาไทย  
4.4 แอปพลิเคชันต้องสามารถใช้งานได้ตลอด 24 ชั่วโมง  
### 5. ข้อกำหนดคุณสมบัติด้านการใช้งาน (Non-Functional Requirements)  
5.1 ระบบต้องทำงานได้รวดเร็ว กดแล้วเปลี่ยนหน้าทันที ไม่เกิดอาการค้างหรือหน่วง  
5.2 ตัวอักษรและปุ่มกดต้องมีขนาดเหมาะสม มองเห็นชัดเจน และใช้งานง่ายแม้สำหรับผู้สูงอายุ  
5.3 ระบบต้องมีความเสถียร และสามารถรองรับผู้ใช้งานจำนวนมากพร้อมกันได้  
5.4 ระบบต้องมีความปลอดภัยของข้อมูลผู้ใช้งาน โดยป้องกันการเข้าถึงจากผู้ที่ไม่ได้รับอนุญาต  
### 6. ข้อกำหนดเพิ่มเติม  
6.1 ระบบต้องสามารถพัฒนาและขยายฟังก์ชันในอนาคตได้ เช่น  
-ระบบชำระเงินออนไลน์  
-ระบบวิเคราะห์การใช้งานที่จอดรถ
